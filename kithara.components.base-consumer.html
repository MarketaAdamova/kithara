Options:

wrap

(wrap message-handler consumer-name & [{:keys [auto-ack? default error-default], :or {default {:ack? true}, error-default {:nack? true}}, :as opts}])

Wrap the given function, taking a kithara message map, to ACK/NACK/REJECT based on the return value:

  • {:reject? true, :requeue? <bool>} -> REJECT (defaults to no requeue),
  • {:nack? true, :requeue? <bool>} -> NACK (defaults to requeue),
  • {:ack? true}-> ACK,
  • {:done? true} -> do nothing (was handled directly).

Any non-map value (plus those without any of the flags set) will be interpreted as default.

Additionally, the following keys can be given:

  • :message: a message to log,
  • :error: an exception to log.
